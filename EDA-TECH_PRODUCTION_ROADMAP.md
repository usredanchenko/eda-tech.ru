# 🚀 EDA-TECH Production Roadmap
## Промпт-Dependency Map для доработки сайта eda-tech.ru

### 📋 Executive Summary

**Статус проекта:** Готов к доработке для продакшн
**Основные проблемы:** 
- Неполная локализация (60% переведено)
- Стандартные системные шрифты, не оптимизированные для кириллицы
- Отсутствие производственных конфигураций

**Цель:** Создать полноценный русскоязычный сайт с красивой кириллической типографикой

---

## 🏗️ Архитектура проекта

### 📱 Экраны и компоненты
1. **Header** - Навигационная панель с языковым переключателем
2. **Hero** - Главный экран с презентацией компании
3. **Services** - Описание услуг (веб, мобайл, боты)
4. **Portfolio** - Портфолио проектов
5. **History** - История компании с timeline
6. **Contact** - Контактная форма и способы связи
7. **Footer** - Подвал с ссылками и соцсетями
8. **OrderForm** - Многошаговая форма заказа (4 шага)
9. **OrderSuccess** - Страница успешного заказа

### 🔧 Текущий tech stack
- **Frontend:** React 18, TypeScript, TailwindCSS
- **Анимации:** Motion/React (Framer Motion)
- **UI:** Radix UI + shadcn/ui
- **Сборка:** Vite
- **Локализация:** Кастомный хук useLanguage

## 🎯 Промпт-Dependency Map

### Phase 1: Основа (Foundation) 🏢
**Роль:** Senior Frontend Developer + UX Designer

#### Prompt 1.1: Анализ и аудит локализации
```
Роль: Senior i18n Specialist

Задача: Провести полный аудит локализации сайта EDA-TECH

Действия:
1. Проанализировать файл useLanguage.tsx 
2. Найти все hardcoded английские тексты в компонентах
3. Создать список недостающих переводов
4. Выявить компоненты, не использующие систему локализации
5. Предложить стратегию полной русификации

Контекст: Сайт студии разработки, целевая аудитория - русскоязычные клиенты
Приоритет: Критический
```

#### Prompt 1.2: Анализ типографики и шрифтов
```
Роль: Typography Specialist + UI Designer

Задача: Оптимизировать типографику для кириллицы

Анализ:
1. Текущие шрифты: системные (ui-sans-serif, system-ui)
2. Проблема: стандартные шрифты выглядят скучно для кириллицы
3. Нужны: красивые, читаемые шрифты для русского языка

Решение:
1. Подобрать Google Fonts оптимизированные для кириллицы
2. Настроить font-display: swap для производительности
3. Создать типографическую иерархию
4. Обеспечить поддержку всех русских символов

Рекомендуемые шрифты: Inter, Manrope, Onest, или другие современные
```

### Phase 2: Контент и локализация (Content) 📝
**Роль:** Content Manager + Translator

#### Prompt 2.1: Доработка системы переводов
```
Роль: Senior Frontend Developer + i18n Specialist

Задача: Завершить локализацию всех компонентов

План:
1. Добавить недостающие ключи переводов в translations объект
2. Заменить hardcoded тексты на вызовы t() функции
3. Проверить корректность всех русских переводов
4. Оптимизировать структуру переводов
5. Добавить fallback для отсутствующих переводов

Компоненты для доработки:
- Contact.tsx (частично не переведен)
- Footer.tsx (захардкожены тексты)
- History.tsx (статистика и описания)
- OrderForm.tsx (некоторые подписи)

Цель: 100% локализация интерфейса
```

## 🚨 Риски и митигация

### Высокие риски
1. **Ломка дизайна при изменении шрифтов**
   - Митигация: тестировать на всех экранах поэтапно

2. **Production ошибки при деплое**
   - Митигация: staging окружение для тестирования

3. **SEO падение при изменениях**
   - Митигация: постепенный rollout, мониторинг позиций

### Средние риски
1. **Производительность после добавления шрифтов**
   - Митигация: font-display swap, preloading

2. **Совместимость с разными браузерами**
   - Митигация: полифиллы, fallback шрифты
